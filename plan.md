# Wicked Wings Slaughter — Marketing Site Plan

**Last updated:** May 1, 2026  
**Status:** Blocked on Node 20 install — see [Blocker](#blocker-node-20-required) below before building.

---

## Overview

A **Vite + React + TypeScript marketing SPA** for the Austin sports bar. Ports the four existing Stitch HTML mocks into React + TanStack Router + shadcn/ui, themed with the "Wicked Neon" design system. Conversion goals are **Book a Table**, **Notify Me on event**, and **Contact form** — all wired to a mock API behind a typed `WingsApi` boundary so a real backend can drop in later without touching components.

**No cart, no online ordering, no staff/admin in this build.**

## API Boundary

```ts
// src/api/client.ts
export interface WingsApi {
  getMenu(): Promise<MenuData>;
  listEvents(): Promise<RecurringEvent[]>;
  createReservation(input: ReservationInput): Promise<Reservation>;
  subscribeToEvent(input: { eventId: string; email: string }): Promise<NotifySubscription>;
  submitContactMessage(input: ContactInput): Promise<{ id: string }>;
}

// Swap VITE_API_MODE=http to wire a real backend — zero component changes needed
export const api: WingsApi =
  import.meta.env.VITE_API_MODE === "http" ? httpClient : mockClient;
```

`mockClient` persists all submissions to `localStorage`. Note this in the README so reviewers know data doesn't leave the browser.

## shadcn Rules (quick ref)

1. `gap-*` not `space-*` for spacing between children.
2. `size-*` not `w-* h-*` when width = height.
3. Semantic tokens only (`text-primary`, `bg-card`) — never raw `text-orange-600` or `bg-zinc-900`.
4. No manual `dark:` overrides — the app is permanently dark.
5. Forms: always `FieldGroup` > `Field` > `FieldLabel` + control. Validation: `data-invalid` on `Field`, `aria-invalid` on the control.
6. Every `Dialog` / `Sheet` / `Drawer` needs a `DialogTitle` / `SheetTitle` (use `className="sr-only"` if visually hidden).
7. Buttons with icons: put the icon inside `<Button>` with `data-icon="inline-start"` or `data-icon="inline-end"`, no sizing classes on the icon.
8. Toasts via `sonner` (`import { toast } from "sonner"`), not custom alert divs.
9. No manual `z-index` on overlays — Dialog/Sheet/Sonner manage their own stacking.
